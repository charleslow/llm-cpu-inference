# SPEC.md — cpu-llm-bench

Benchmark harness measuring the **latency vs quality pareto frontier** of small LLMs on CPU. An agent drives exploration — no predefined search space. Generation is decoupled from scoring: all prompts are plain text completions, any framework works.

Your job is to implement this harness.

---

## Project structure

```
program.md              # agent reads each iteration
pyproject.toml          # uv for package management
constants.py            # constants like max tokens, sequence length, temperature etc.
protocol.py             # Protocols / Contracts that backend needs to obey
setup.py                # One time setup script
run_backend.py          # loads backends/backend.py, runs generation loop
score.py                # scores completions against gold answers
data/
    prompts.jsonl       # gitignored, generated by setup. Contains prompt and answer.
backends/
    hf.py               # seed implementation
    backend.py          # Agent to overwrite and implement in each loop
completions/            # gitignored
results/
    experiment.log      # gitignored, full logs from last run for debugging
    summary.jsonl       # the frontier — append-only experiment log
    analyses/           # one markdown file per iteration
run.sh                  # Orchestrates the agentic loop via claude -p
```

## Agent loop

`run.sh` orchestrates a fixed number of iterations (default 20). Each iteration has two `claude -p` phases:

**Phase 1 — Analysis** (claude -p with web search):
1. Read `program.md`, `results/summary.jsonl`, and (if last run failed) `results/experiment.log`
2. Research: search the web for model options, GGUF repos, framework benchmarks, quantization docs
3. Decide what to try next and why
4. Write `results/analyses/trial-{N}.md` — a short markdown doc with:
   - What was tried before and results so far
   - What this iteration will try and the hypothesis
   - Specific model name, framework, quantization, and any install steps

**Phase 2 — Implementation** (claude -p):
5. Read the analysis file and `program.md`
6. Implement `backends/backend.py` (the only file the agent modifies)
7. **Dry run**: `python run_backend.py --dry-run` — runs 3 prompts, validates output format, prints latency. Abort and fix if this fails.

Then `run.sh` takes over:
8. **Full run**: `python run_backend.py` — runs all prompts. Hard **10-minute wall-clock timeout**; killed if exceeded (status: `timeout`).
9. **Score**: `python score.py` — scores the latest completions, writes `results/trial-{N}-{name}.jsonl`
10. **Log**: append a summary line to `results/summary.jsonl`
11. Repeat

The only file the agent hacks is `backends/backend.py`. The Pareto frontier (latency vs accuracy) is eyeballed from `results/summary.jsonl` — no automated frontier computation.

## File specifications

### 1. `protocol.py`

```python
@dataclass
class GenerationResult:
    completion: str          # extracted answer text
    raw_output: str          # full model output
    latency_ms: float        # wall-clock time
    tokens_generated: int
    prompt_tokens: int = 0

@runtime_checkable
class InferenceBackend(Protocol):
    def setup(self) -> None: ...
    def generate(self, prompt: str, max_tokens: int = 256) -> GenerationResult: ...
    def teardown(self) -> None: ...
```

- `setup()`: called once
- `generate()`: called per prompt. Measure latency with `time.perf_counter()`.
- `teardown()`: called once after all prompts.

### 3. `setup.py`

~50 questions, seed=42. Pulls from HF datasets directly.
- MMLU: 25%
- ARC-Challenge: 20%
- HellaSwag: 20%
- GSM8K: 15%
- WinoGrande: 10%
- TruthfulQA: 10%

We just need inference to generate the answer (no log probs). Then we score by comparing answer against gold.

**Prompt format per benchmark:**

Each entry in `data/prompts.jsonl` has: `prompt`, `gold`, `task`, `answer_type`.

- **MMLU** (`answer_type: "choice"`): multiple-choice, 4 options A-D. Prompt ends with `"Answer:"`. Gold is a single letter.
- **ARC-Challenge** (`answer_type: "choice"`): multiple-choice, 3-5 options. Same format as MMLU.
- **HellaSwag** (`answer_type: "choice"`): sentence completion, 4 options A-D. Prompt presents the context + numbered completions, ends with `"Answer:"`. Gold is a single letter.
- **GSM8K** (`answer_type: "numeric"`): math word problem. Prompt ends with `"Answer:"`. Gold is a number (extracted via `#### N` pattern).
- **WinoGrande** (`answer_type: "choice"`): binary choice (A or B). Prompt presents a sentence with a blank and two options, ends with `"Answer:"`. Gold is A or B.
- **TruthfulQA** (`answer_type: "choice"`): multiple-choice, 4 options A-D. Same format as MMLU.

### 4. `run_backend.py`

**CLI:**
```
python run_backend.py              # full run (all prompts)
python run_backend.py --dry-run    # 3 prompts, validate output, print latency
```
- Loads `backends/backend.py`, validates it against the `InferenceBackend` protocol
- Generates a `run_id` of the form `trial-{N}-{name}` where:
  - `{N}` = auto-incremented trial number (counts existing `results/trial-*.jsonl` files + 1)
  - `{name}` = slugified from a `TRIAL_NAME` constant that the agent sets in `backend.py`
- Outputs completions to `completions/{run_id}.jsonl`
- In `--dry-run` mode: runs only `DRY_RUN_PROMPTS` prompts, prints per-prompt latency, exits 0 on success
- Enforced `TIMEOUT_SECONDS` wall-clock limit (full run only); killed with `SIGTERM` if exceeded

### 5. `score.py`

**CLI:** `python score.py`

Scores the latest completion and produce accuracy scores.

**Answer extraction:**

`extract_choice(raw_output)`:
1. Stripped output is single letter A-E → return it
2. Pattern `(?:answer|correct)\s*(?:is|:)\s*([A-E])` (case insensitive)
3. First standalone `\b([A-E])\b`
4. First character uppercased

`extract_numeric(raw_output)`:
1. Pattern `####\s*(.+?)(?:\s|$)`
2. Pattern `(?:answer|result)\s*(?:is|=|:)\s*([\d,.-]+)`
3. Last number: `(-?[\d,]+\.?\d*)`
4. Strip commas

**Scoring**: choice → `predicted == gold`. Numeric → compare as floats, fallback string compare.

Outputs:
- `results/trial-{N}-{name}.jsonl` — per-prompt results (prompt, gold, predicted, correct, latency_ms)
- Appends one line to `results/summary.jsonl` (see schema below)

### 9. `run.sh`

Orchestrates the full agentic loop. Accepts an iteration count argument.

**CLI:**
```bash
./run.sh              # default: 20 iterations
./run.sh 50           # run 50 iterations
```

Pseudocode:
```bash
#!/usr/bin/env bash
set -e
ITERATIONS=${1:-20}
TRIAL_NUM=$(ls results/trial-*.jsonl 2>/dev/null | wc -l)

for i in $(seq 1 $ITERATIONS); do
    TRIAL_NUM=$((TRIAL_NUM + 1))
    echo "=== Iteration $i / $ITERATIONS (trial $TRIAL_NUM) ==="

    # Phase 1: Analysis — research and plan
    claude -p "You are the analysis agent. Read program.md and results/summary.jsonl. \
        Research what to try next (web search allowed). \
        Write results/analyses/trial-${TRIAL_NUM}.md with your plan." \
        --allowedTools "Edit,Read,Bash,WebSearch" \
        2>&1 | tee results/experiment.log

    # Phase 2: Implementation — code the backend
    claude -p "You are the implementation agent. Read program.md and \
        results/analyses/trial-${TRIAL_NUM}.md. \
        Implement backends/backend.py. Run the dry-run to validate." \
        --allowedTools "Edit,Read,Bash" \
        2>&1 | tee -a results/experiment.log

    # Belt-and-suspenders dry run
    timeout 120 python run_backend.py --dry-run 2>&1 | tee -a results/experiment.log
    if [ $? -ne 0 ]; then echo "Dry run failed"; continue; fi

    # Full run with hard timeout
    timeout 600 python run_backend.py 2>&1 | tee -a results/experiment.log
    EXIT_CODE=$?
    if [ $EXIT_CODE -eq 124 ]; then
        # timeout(1) returns 124 when the command is killed
        python score.py --status timeout 2>&1 | tee -a results/experiment.log || true
        continue
    elif [ $EXIT_CODE -ne 0 ]; then
        python score.py --status crash 2>&1 | tee -a results/experiment.log || true
        continue
    fi

    # Score
    python score.py 2>&1 | tee -a results/experiment.log || { echo "Scoring failed"; }
done
```

**Phase 1** (analysis) has web search access and produces a human-readable plan in `results/analyses/`. **Phase 2** (implementation) reads the plan and writes code. The outer loop handles the full run, timeout detection, and scoring.

### 11. `program.md`

Contains: mission statement, per-iteration loop, backend contract reference, and rules.
- Debug and retry failures; log them. Skip uninstallable frameworks.
- Must set `TRIAL_NAME` in `backend.py` to a descriptive slug each iteration
